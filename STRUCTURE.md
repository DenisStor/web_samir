# Структура проекта Say's Barbers

## Обзор

Проект был отрефакторен для удобства разработки и поддержки. Код разделен на логические модули.

## Структура файлов

```
web_samir/
├── index.html              # Основной HTML файл
├── server.py              # Простой HTTP сервер для локальной разработки
├── css/                   # CSS модули
│   ├── variables.css      # CSS переменные (цвета, размеры)
│   ├── base.css          # Базовые стили (reset, scrollbar, typography)
│   ├── navigation.css     # Стили навигации и меню
│   ├── components.css     # Компоненты (кнопки, формы, бейджи)
│   └── sections.css       # Стили всех секций сайта
├── js/                    # JavaScript модули
│   ├── navigation.js      # Навигация и мобильное меню
│   ├── animations.js      # Анимации и scroll эффекты
│   ├── forms.js          # Формы и валидация
│   └── modals.js         # Модальные окна и FAQ
└── assets/               # Статические ресурсы (изображения)

# Архивные файлы
├── says_barbers.html      # Оригинальный HTML файл (backup)
└── says_barbers.py        # Старый скрипт генерации
```

## CSS Модули

### variables.css (25 строк)
Содержит все CSS переменные:
- Цветовая палитра (темная тема, акценты)
- Цвета текста
- Цвета бейджей
- Медицинская секция (светлая тема)

### base.css (64 строки)
Базовые стили:
- CSS reset (*, html, body)
- Кастомный scrollbar
- Типография (заголовки, шрифты)
- Container класс
- Fade-in анимации

### navigation.css (251 строка)
Навигация:
- Фиксированная навигация (.nav)
- Логотип
- Навигационные ссылки
- Мобильное меню с бургером
- Marquee секция
- Адаптивные стили

### components.css (225 строк)
Переиспользуемые компоненты:
- Кнопки (primary, secondary, submit)
- Бейджи и теги
- Price tags
- Формы (inputs, selects, textarea)
- Success сообщения

### sections.css (1,189 строк)
Все секции сайта:
- Hero секция с анимациями
- Services (услуги)
- Podology (подология)
- Masters (команда)
- Quality (качество)
- Location (локация)
- Social (Instagram)
- Blog с модальными окнами
- FAQ (аккордеон)
- Booking (форма записи)
- Footer

## JavaScript Модули

### navigation.js
**Функции:**
- `toggleMenu()` - Переключение мобильного меню
- `closeMenu()` - Закрытие меню
- Scroll эффект для навигации

### animations.js
**Функции:**
- IntersectionObserver для fade-in анимаций
- Автоматическое появление элементов при скролле

### forms.js
**Функции:**
- `submitForm(e)` - Обработка отправки формы
- Установка минимальной даты для date input
- Маска телефона в формате +7 (XXX) XXX-XX-XX

### modals.js
**Функции:**
- `openBlogModal(articleId)` - Открытие модального окна блога
- `closeBlogModal()` - Закрытие модального окна
- `toggleFaq(element)` - Переключение FAQ элементов
- Закрытие по клику вне окна и по ESC

## Запуск проекта

### Способ 1: Python сервер (рекомендуется)
```bash
python3 server.py
```
Сервер запустится на http://localhost:8000 и автоматически откроет браузер.

### Способ 2: Любой другой HTTP сервер
```bash
# Python SimpleHTTPServer
python3 -m http.server 8000

# Node.js http-server
npx http-server

# PHP встроенный сервер
php -S localhost:8000
```

## Как редактировать сайт

### Изменение цветов
Отредактируйте `css/variables.css`:
```css
:root {
    --accent-green: #00ff88;  /* Изменить основной акцентный цвет */
    --bg-dark: #0d0d0d;       /* Изменить фон */
}
```

### Изменение стилей кнопок
Отредактируйте `css/components.css`:
```css
.btn-primary {
    /* Ваши изменения */
}
```

### Изменение конкретной секции
Найдите нужную секцию в `css/sections.css`:
```css
/* Hero Section */
.hero {
    /* Ваши изменения */
}
```

### Изменение логики форм
Отредактируйте `js/forms.js`:
```javascript
function submitForm(e) {
    // Ваша логика
}
```

## Порядок подключения файлов

### CSS (в &lt;head&gt;)
1. variables.css (переменные должны быть первыми)
2. base.css
3. navigation.css
4. components.css
5. sections.css

### JavaScript (перед &lt;/body&gt;)
1. navigation.js
2. animations.js
3. forms.js
4. modals.js

**Важно:** Порядок подключения имеет значение! Не меняйте его.

## Преимущества новой структуры

1. **Модульность** - легко найти и изменить нужный компонент
2. **Переиспользование** - компоненты можно использовать в других проектах
3. **Совместная работа** - разные разработчики могут работать над разными модулями
4. **Производительность** - браузер кэширует отдельные файлы
5. **Отладка** - легче найти ошибки в небольших файлах
6. **Гибкость** - можно отключить ненужные модули
